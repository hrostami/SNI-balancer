# xray-balancer

یک بالانسر هوشمند و خودکار برای پروکسی Xray. این ابزار لیستی از کانفیگ‌های VLESS و Trojan را با اندازه‌گیری سرعت دانلود واقعی و تأخیر شبکه تست می‌کند، سپس به‌صورت خودکار بهترین کانفیگ را به‌عنوان یک پروکسی SOCKS5 اجرا می‌کند که از هر دستگاهی در شبکه محلی (LAN) قابل دسترس است.

هیچ وابستگی به v2rayN یا هیچ کلاینت دیگری ندارد.

## نحوه کار

1. کانفیگ‌ها از فایل `configs.txt` خوانده می‌شوند — یک URI در هر خط، یا لینک سابسکریپشن
2. برای هر کانفیگ، یک نمونه موقت Xray روی یک پورت اختصاصی اجرا می‌شود
3. تست دو مرحله‌ای انجام می‌شود: بررسی سلامت (تأخیر) → تست سرعت دانلود
4. هر کانفیگ بر اساس ترکیبی وزن‌دار از سرعت، تأخیر، و پایداری تاریخچه‌ای امتیازدهی می‌شود
5. یک پروکسی SOCKS5 پایدار با بهترین کانفیگ راه‌اندازی می‌شود
6. در فاصله‌های زمانی قابل تنظیم تست تکرار می‌شود؛ تنها در صورتی سوئیچ می‌کند که بهبود امتیاز از یک آستانه مشخص بیشتر باشد
7. تاریخچه تست‌ها در فایل `config_history.json` ذخیره می‌شود و پس از ریستارت باقی می‌ماند

## پیش‌نیازها

- Python نسخه 3.9 یا بالاتر
- `curl` (روی macOS و اکثر توزیع‌های Linux از پیش نصب است؛ برای ویندوز از [curl.se](https://curl.se/windows/) دانلود کنید)
- باینری Xray (مراحل نصب را در ادامه ببینید)

## راه‌اندازی

**۱. کلون کردن ریپازیتوری**
```bash
git clone https://github.com/yourname/xray-balancer.git
cd xray-balancer
```

**۲. نصب وابستگی‌های Python**
```bash
pip3 install -r requirements.txt
```

**۳. افزودن باینری Xray**

باینری Xray مناسب سیستم‌عامل خود را از [صفحه ریلیزهای XTLS/Xray-core](https://github.com/XTLS/Xray-core/releases) دانلود کرده و در پوشه پروژه قرار دهید:

```bash
# macOS / Linux
cp /path/to/xray ./xray
chmod +x ./xray

# ویندوز: فایل xray.exe را در پوشه پروژه قرار دهید
```

**۴. ساخت فایل `configs.txt`**

کانفیگ‌های پروکسی خود را در این فایل قرار دهید، یک URI در هر خط. فرمت‌های `vless://` و `trojan://` و همچنین لینک‌های سابسکریپشن (متن ساده یا base64) پشتیبانی می‌شوند:

```
vless://uuid@host:port?...#نام
trojan://password@host:port?...#نام
https://آدرس-سابسکریپشن-شما.com/sub
```

خطوط خالی نادیده گرفته می‌شوند. آدرس و پورت داخل URI به‌صورت خودکار به `127.0.0.1:40443` تبدیل می‌شوند.

## نحوه استفاده

**تست همه کانفیگ‌ها بدون راه‌اندازی Xray:**
```bash
python3 balancer.py --dry-run
```

**اجرای معمولی:**
```bash
python3 balancer.py
```

**تنظیم فاصله زمانی بین تست‌ها (مثلاً هر ۵ دقیقه):**
```bash
python3 balancer.py --interval 300
```

**تغییر پورت پروکسی SOCKS5:**
```bash
python3 balancer.py --port 1080
```

**افزایش حجم نمونه دانلود (دقیق‌تر، کندتر):**
```bash
python3 balancer.py --test-size 10
```

**مسیر دلخواه برای فایل کانفیگ‌ها:**
```bash
python3 balancer.py --configs /path/to/myconfigs.txt
```

**همه گزینه‌ها:**
```
--dry-run           تست کانفیگ‌ها و نمایش نتایج، بدون راه‌اندازی Xray
--interval N        فاصله زمانی بین چرخه‌های تست به ثانیه (پیش‌فرض: 1800)
--port N            پورت پروکسی SOCKS5 (پیش‌فرض: 4567)
--configs PATH      مسیر فایل کانفیگ‌ها (پیش‌فرض: configs.txt در پوشه اسکریپت)
--test-size N       حجم نمونه تست دانلود به مگابایت (پیش‌فرض: 1)
--display-time N    مدت نمایش جدول نتایج کامل به ثانیه (پیش‌فرض: 5)
```

## اتصال سایر دستگاه‌ها

پس از اجرا، هر دستگاهی در شبکه محلی می‌تواند از پروکسی استفاده کند:
```
پروتکل : SOCKS5
هاست   : <آی‌پی LAN دستگاه شما>
پورت   : 4567  (یا هر پورتی که با --port تنظیم کردید)
```

برای یافتن آی‌پی LAN دستگاه خود:
```bash
# macOS / Linux
ip addr show   # یا: ifconfig

# ویندوز
ipconfig
```

## سیستم امتیازدهی

هر کانفیگ بر اساس سه فاکتور وزن‌دار از تاریخچه تست‌های اخیر امتیاز می‌گیرد:

| فاکتور | وزن | توضیح |
|---|---|---|
| سرعت | ۴۰٪ | میانگین سرعت دانلود در تست‌های موفق اخیر |
| تأخیر | ۳۰٪ | میانگین زمان رفت‌وبرگشت در بررسی‌های سلامت |
| پایداری | ۳۰٪ | نسبت تست‌های موفق در پنجره تاریخچه |

سوئیچ تنها زمانی انجام می‌شود که امتیاز بهترین کانفیگ جدید از امتیاز کانفیگ فعلی بیشتر از `SWITCH_THRESHOLD` (پیش‌فرض: ۲۰٪) باشد. این از جابجایی مکرر بین کانفیگ‌های هم‌سطح جلوگیری می‌کند.

کانفیگ‌هایی که بارها شکست می‌خورند مشمول backoff نمایی می‌شوند — برای مدت‌های فزاینده‌ای از تست حذف می‌شوند تا از اتلاف وقت جلوگیری شود.

## پروتکل‌های پشتیبانی‌شده

| پروتکل | ترنسپورت‌ها |
|---|---|
| `vless` | WebSocket، xHTTP، gRPC، HTTPUpgrade، SplitHTTP |
| `trojan` | WebSocket، xHTTP، gRPC، HTTPUpgrade، SplitHTTP |

حالت‌های امنیتی TLS و REALITY پشتیبانی می‌شوند. Flow control برای VLESS (مثل `xtls-rprx-vision`) نیز پشتیبانی می‌شود.

## ثابت‌های قابل تنظیم

تمام مقادیر قابل تنظیم در ابتدای فایل `balancer.py` قرار دارند:

| ثابت | پیش‌فرض | توضیح |
|---|---|---|
| `SOCKS_PORT` | `4567` | پورت پروکسی SOCKS5 |
| `SOCKS_LISTEN` | `0.0.0.0` | آدرس شنود (`0.0.0.0` = همه اینترفیس‌ها) |
| `SNI_PORT` | `40443` | پورت تونل upstream |
| `TEST_URL` | Cloudflare 1MB | آدرس مورد استفاده برای تست سرعت دانلود |
| `HEALTH_URL` | gstatic 204 | آدرس مورد استفاده برای بررسی سلامت |
| `TEST_TIMEOUT` | `15` | حداکثر ثانیه برای تست دانلود |
| `HEALTH_TIMEOUT` | `5` | حداکثر ثانیه برای بررسی سلامت |
| `CHECK_INTERVAL` | `1800` | فاصله زمانی بین چرخه‌های تست به ثانیه |
| `BASE_TEST_PORT` | `19000` | پورت شروع برای نمونه‌های موقت تست |
| `W_SPEED` | `0.4` | وزن سرعت در امتیازدهی |
| `W_STABILITY` | `0.6` | وزن پایداری در امتیازدهی |
| `SWITCH_THRESHOLD` | `0.2` | حداقل بهبود امتیاز لازم برای سوئیچ |
| `HISTORY_WINDOW` | `6` | تعداد نتایج اخیر در نظر گرفته‌شده برای امتیازدهی |
| `DISPLAY_TOP_N_FULL` | `10` | تعداد کانفیگ‌های نمایش‌داده‌شده در جدول کامل |
| `DISPLAY_TOP_N_COMPACT` | `3` | تعداد کانفیگ‌های نمایش‌داده‌شده در حالت فشرده |
| `PING_COUNT` | `3` | تعداد پینگ برای میانگین‌گیری تأخیر |
| `XRAY_STARTUP_WAIT` | `2.0` | ثانیه انتظار پس از راه‌اندازی Xray |

## فایل‌ها

| فایل | توضیح |
|---|---|
| `balancer.py` | اسکریپت اصلی |
| `configs.txt` | کانفیگ‌های پروکسی شما (کامیت نمی‌شود) |
| `xray` / `xray.exe` | باینری Xray (کامیت نمی‌شود) |
| `xrayconfig.json` | کانفیگ Xray فعال تولیدشده (کامیت نمی‌شود) |
| `config_history.json` | تاریخچه تست‌های ذخیره‌شده (کامیت نمی‌شود) |
| `balancer.log` | لاگ اجرا (کامیت نمی‌شود) |

## نکات

- برای خروج از `Ctrl+C` استفاده کنید — Xray همیشه در هنگام خروج، حتی در صورت کرش، متوقف می‌شود
- `configs.txt`، باینری Xray، و تمام فایل‌های تولیدشده در `.gitignore` هستند — هرگز کانفیگ‌های خود را کامیت نکنید
- روی ویندوز مطمئن شوید که `curl` در PATH سیستم شما قرار دارد
- Python نسخه 3.9 یا بالاتر الزامی است
